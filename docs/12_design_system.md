# ⑫ デザインシステム — HydroPaw (Flutter / Web 共通)

Flutter(`app/lib/core/theme/app_theme.dart`)とWeb(`webapp/src/styles.css`)は
同一トークンを実装する。片方を変えるときは必ず両方を更新する。

## 1. 世界観

「病気を探す装置」ではなく「毎日そっと見守る相棒」。
開いた瞬間に伝えるのは 安心・優しさ・静けさ。
言葉は責めない・驚かせない(「高めです。様子を見ましょう」であって
「異常値です」ではない)。絵文字は使わず、線の細いアイコンだけを使う。

| textSecondary | rgba(60,60,67,.6) | rgba(235,235,245,.6) | 補足 |
| textTertiary | rgba(60,60,67,.35) | rgba(235,235,245,.3) | キャプション |

グラデーション・ドロップシャドウは原則禁止(カードの極薄シャドウのみ)。

## 3. タイポグラフィ

| 役割 | サイズ/ウェイト/字間 | 例 |
|---|---|---|
| LargeTitle | 28–34 / 700 / -0.6px | 画面タイトル・犬の名前 |
| 状態フレーズ | 24–26 / 700 / -0.5px | 「今日は安定しています」 |
| Title | 19–22 / 600 / -0.3px | カード見出し級 |
| Body | 15–16 / 400–500 / 1.5行間 | コメント |
| Caption | 12–13 / 500 | 時刻・単位・補足 |
| Overline | 11–12 / 600 / +0.8px 大文字 | セクションラベル |
| 数値 | tabular-nums必須 | ppm・時刻・統計 |

フォントはシステム(SF Pro / Roboto / Hiragino)。ウェイトと色の階層で
情報の優先順位を作り、サイズの種類は増やさない。

## 4. コンポーネント

- **カード**: 角丸16–18 / ヘアライン(darkのみ) / 極薄シャドウ(lightのみ) /
  padding 18–24。カードを増やすより余白を増やす。
- **主ボタン**: 高さ50–54 / 角丸14(Web)・27(Flutter pill) / accent塗り。
  測定中の「終了する」だけは textPrimary 塗り(行為の重さを色で変える)。
- **状態ピル**: 12px/600、状態色の10%面+同色文字、ドット6px。
- **状態ドット**: 8–11px円。状態色の唯一の原色使用箇所。
- **タブバー**: モバイル=下部固定+ブラー背景、iPad/Mac=ヘッダ内セグメント。
  アイコン20–23px(stroke 1.8)+10.5–13pxラベル。選択=accent。
- **入力**: cardElevated面 / 角丸12–14 / フォーカスはaccentリング1.5px。
- **アイコン**: SF Symbols相当の線画のみ(Flutter=CupertinoIcons,
  Web=自作SVG 24グリッド/stroke1.8)。絵文字禁止。

## 5. モーション

| 場面 | 仕様 |
|---|---|
| 画面遷移 | fade + scale 0.97→1.0 / 320ms / easeOutCubic |
| ビュー切替(Web) | fade + 6px上昇 / 320ms |
| 呼吸リング | scale 1→1.04 / 4s / ease-in-out 無限(待機時は6s) |
| 状態語の変化 | クロスフェード 250–300ms |
| 数値バー | 高さ 500ms easeOutCubic |
| 完了バッジ | pop 0.92→1 / 500ms |
| 触覚(Flutter) | 開始=light / 完了=medium impact |

原則: 動きは「状態が変わったこと」を伝えるためだけに使う。装飾のループ
アニメーションは呼吸リングのみ。`prefers-reduced-motion` を尊重する。

## 6. 言葉(トーン)

- 主語は犬。「ポチの呼気を測定」「今日は安定しています」。
- 命令しない: 「〜しましょう」「〜をおすすめします」。
- 専門用語(ppm/BLE/Mock/UUID)はホームに出さない。居場所は
  履歴詳細・結果の脚注・設定。
- 解析の間(1.2–1.4s)を意図的に置き、測定を「儀式」にする。

## 7. 情報設計の原則 (Apple Health等から移植した思想)

1. **1画面1つの問いに答える**: ホーム=「今日は元気?」、結果=「どうだった?」。
2. **数値は要求されたときに**: タップで深くなるほど専門的に。
3. **スクロールは1カラム**: グリッドダッシュボード化しない。
4. **空状態も体験**: 「まだ測定がありません」ではなく「はじめての測定をしてみましょう」。
5. **技術の隔離**: 接続・電池・データソースは設定タブのみ。
