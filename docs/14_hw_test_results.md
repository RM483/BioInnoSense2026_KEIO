# ⑭ 実機テスト結果記録 (docs/13の実施記録)

- 状態は **PASS / FAIL / 保留 / 未確認** の4値のみ。
- 実測結果が得られるまで「未確認」を維持する(推測でPASSにしない)。
- FAIL時: 問題内容→修正コミット→再テスト結果 の順に必ず埋める。
- ログ抜粋はLPUART1の該当行をそのまま貼る。

## 数値の区分 (必須)

本ファイルに書く全ての数値には次の区分を付ける:

- **MEASURED** — 実機またはビルド成果物から取得した値
- **ESTIMATED** — コード・仕様からの推定値
- **UNKNOWN** — 確認できていない値

**ESTIMATED / UNKNOWN はPASS判定の根拠にできない。** 実測が完了する
まで該当テストは「未確認」を維持する。

## 進行ルール

1. **1テストずつ**実施し、完了ごとに本ファイルへ記録する。
2. FAIL時は**その場で修正しない**。まず再現条件とログを固定する。
3. 同一条件で**最低3回**再現を確認してから原因調査に入る。
4. 原因仮説を**複数**列挙し、最小の切り分けテストで絞り込む。
5. 原因が特定できてから修正する(予防的修正・ついで修正は禁止)。
6. 修正後は対象テストに加え、**関連テストも再実施**する
   (例: power.c修正→T0/T1/T6/T8を再走)。
7. P0/P1が全てPASSするまでP2(STOP2系)/P3(ソーク等)へ進まない。
