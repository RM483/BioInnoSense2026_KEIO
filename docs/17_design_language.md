# ⑰ HydroPaw Design Language v1.0

策定: 2026-07-18。実装より先に本書を確定した。
docs/12(トークン)・docs/16(UX)の上位文書。矛盾時は本書が優先。

## 1. Brand Concept

**HydroPawとは**: 犬の呼気に含まれる水素から腸のコンディションを読み取り、
「ことばを持たない家族」の体調を、飼い主のことばにして毎日届けるプロダクト。

**ブランドアイデア**:
> **ことばにならないものを、ことばに。**

犬は「お腹の調子が悪い」と言えない。HydroPawは呼吸ひとつからそれを聞き取り、
「今日は安定しています」という一文に翻訳する。センサーでも、グラフでも、
数値でもなく、**翻訳者**であること。これがHydroPawのすべての設計判断の根。

**提供価値**: ①毎日30秒の安心 ②変化の早期の気づき ③受診時の確かな記録。
**与える感情**: 安堵 → 習慣 → 信頼。驚かせない。急かさない。責めない。
**世界観**: 朝の散歩のあとの、静かな台所。医療機器ではなく、家族の道具。

## 2. Brand Personality

| である | でない |
|---|---|
| 安心させる(まず結論、まず大丈夫) | 不安を煽る(赤・警告・ビックリマーク) |
| 静か(余白・小声のUI) | 賑やか(バッジ・通知の洪水) |
| 優しい(責めない言葉) | 命令する(「〜してください」の連打) |
| 誠実(わからない時はわからないと言う) | 断定する(診断めいた言い切り) |
| 犬への愛情(主語は犬・写真が中心) | データへの愛情(数値が主役) |
| 医療の裏付け(正常範囲・受診目安) | 医療の威圧(専門用語・臨床UI) |

**声のトーン**: 信頼できる獣医助手が、家族として隣にいる声。
「〜しましょう」「〜そうです」「〜をおすすめします」。決して「〜しなさい」
「危険」「異常」。

## 3. Color System — 「水面(みなも)」パレット

HydroPawの固有色は**Mizuha(水羽) — 深い水緑**。
「Hydro(水)」と「生命(緑)」の交点。健康アプリの定番(赤・青・紫)を避け、
1秒でHydroPawと分かる色。信頼(深さ)と安心(緑)を同時に持つ。

| トークン | Light | Dark | 意味 |
|---|---|---|---|
| **Primary (Mizuha)** | #12766B | #4FB8A5 | ブランドの声。CTA・アクティブ・リング面・ロゴ |
| Primary Soft | Mizuha 10% | 14% | ブランドの気配(面・チップ) |
| Background | #F6F5F2 (紙) | #000000 | 台所のテーブル。医療の白ではない |
| Surface (Card) | #FFFFFF | #1C1C1E | 清潔。信頼は白い面に載せる |
| Surface Elevated | #F0EFEA | #2C2C2E | 入力・チップの面 |
| Success (安定) | #2F9E6E | #46C68A | 「今日も大丈夫」の緑 |
| Warning (注意) | #C8811F | #E8A94E | 「様子を見て」の琥珀。赤ではない |
| Danger (受診目安) | #CE544C | #E9736C | 珊瑚色。血の赤・エラーの赤を避ける |
| Text Primary | #1C1C1E | #F2F2F7 | 結論の色 |
| Text Secondary | rgba(60,60,67,.6) | rgba(235,235,245,.6) | 補足の色 |
| Text Tertiary | rgba(60,60,67,.35) | rgba(235,235,245,.3) | 気配の色 |
| Divider | rgba(60,60,67,.12) | rgba(84,84,88,.5) | 1pxのヘアラインのみ |

**運用律**:
- 状態色(Success/Warning/Danger)が現れてよいのは**4箇所だけ**:
  StatusRing / 状態ドット / チップ / グラフ最新点。
- Dangerは「受診の目安」の意味でのみ使う。エラー・失敗の表現には使わない。
- Primaryは1画面に原則1回(CTAかリング)。ブランドは連呼しない。

## 4. Typography

システムフォント(SF Pro / Hiragino / Roboto)。書体ではなく**組み**で個性を出す。

| 役割 | 仕様 | 例 |
|---|---|---|
| 状態フレーズ | 22–24 / W700 / 字間-0.5 / 行間1.35 | 「今日は安定しています」 |
| LargeTitle | 28 / W700 / -0.6 | タブの題 |
| Title | 19 / W600 / -0.3 | カード見出し |
| Body | 15–16 / W400–500 / 行間1.6(和文) | 行動の一文 |
| Caption | 13 / W500 | 時刻・変化 |
| Overline | 12 / W600 / +0.8 | セクション札 |
| ボタン | 17 / W600 | CTA |
| **数値** | **W600(700にしない)** / tabular-nums必須 / 単位は60%サイズ・Tertiary色で数値の右に半角スペース | 6.2 ppm |

数値の思想: 数値は「証拠」であって「主張」ではない。だから太字にしすぎない、
単位を小さく添える、そして**ホームには置かない**。

## 5. Iconography

- 線画のみ。**stroke 1.8 / 24pxグリッド / 角丸線端(round cap/join)**。
- サイズ: タブ22–23 / 行内18–21 / 補助14–16。タップ標的は最小44×44。
- 塗りは選択状態のタブと肉球マークのみ。絵文字は全面禁止。
- **肉球(Paw)はブランドマーク**: 3指+パッドの固有形状(components参照)を
  アプリアイコン・スプラッシュ・空状態で共通使用。他の肉球素材は使わない。

## 6. Motion — 「呼吸」の物理

すべての動きは犬の呼吸のリズムに従う。装飾のためのアニメーションは禁止。

| 場面 | 仕様 |
|---|---|
| 画面遷移(push) | fade + scale 0.97→1 / 320ms / easeOutCubic(戻り240ms) |
| タブ切替 | 即時(0ms)。タブは場所の移動であって演出ではない |
| Hero (準備→測定) | care-ring連続変形 / 400ms |
| 呼吸リング | scale 1→1.04 / **4s** / easeInOut 無限。待機は6s |
| 状態語の交代 | クロスフェード 250–300ms |
| 完了 | pop 0.92→1 / 500ms + medium haptic |
| Haptics | 開始=light / 完了=medium / タブ・CTA=selection。エラーにheavyは使わない |
| Loading | スピナーは最終手段。原則は呼吸リング(測定)か静かなプレースホルダ |
| reduced-motion | 全ループ停止・遷移はfadeのみ(Web実装済/Flutterは`disableAnimations`尊重) |

| コンポーネント | 律 |
|---|---|
| **StatusRing** | docs/12 §3b。目盛り・%・端点禁止。面はPrimary Soft(写真なし時)、輪は状態色55%、ハロー7% |
| Card | 角丸16(Web)/18(Flutter) / lightは極薄影 / darkはヘアライン / padding 20–24 |
| Button(CTA) | 高さ56–58 / 全丸 / Primary塗り / 押下0.98+selection haptic / 影はPrimary 28% |
| Button(測定終了) | TextPrimary塗り(行為の重さ) |

| Graph | 0基準 / 正常帯(Success 7%)+帯内ラベル / 受診目安は必要時のみ点線 / 線は中立色 / 最新点のみ状態色 |

| Chip | 色+記号(✓/!)+語。色単独禁止 |

## 8. Copy Writing — HydroPawの辞書

主語は犬。時制は現在。文末は「〜です/ます」。禁則と置換:

| 使わない | 使う |
|---|---|
| 危険 / 異常 / 警告 | 少し注意が必要です / 高めの値が続いています |
| エラー / 失敗 | うまく測れませんでした / うまくいきませんでした |
| 接続失敗 | 近くでデバイスを探しています / 見つかったらお知らせします |
| 電池切れ | そろそろ充電をおすすめします |
| 再試行 | もういちど / もういちど測ってみましょう |
| データがありません | はじめての測定をしてみましょう |
| 診断 | 目安 / ようす |
| ローディング中 | 解析しています… / 準備しています… |

数字を語るときも人の言葉を先に:「安定しています(平均 6.2 ppm)」。

## 9. Empty States (すべて「次の一歩」を持つ)

| 状態 | 見せるもの | 一文 | 行動 |
|---|---|---|---|
| 初回起動(犬未登録) | リング+肉球 | 「まずは愛犬のプロフィールを登録しましょう」 | 愛犬を登録する |
| 記録なし | リング(Primary) | 「はじめての測定をしてみましょう」+所要時間の予告 | 測定をはじめる |
| BLE未接続 | 準備リング(中立色) | 「『はじめる』と同時にデバイスへ接続します」 | はじめる/接続 |
| 履歴0件 | 小さな肉球 | 「まだ測定がありません」 | 測定をはじめるへの導線 |
| 検索中 | 静かなスピナー+「デバイスを探しています…」 | — | 自動 |

## 10. Error UX — エラー画面は存在しない

画面の題は状況ではなく**次の一歩**。構成は常に:
やわらかい図(円面+線画) → 何が起きたか(1文・非難なし) → 主ボタン(次の一歩)
→ 副リンク(ホームへ)。

| 状況 | 題 | 一文 | 主ボタン |
|---|---|---|---|
| センサ無応答 | もういちど測ってみましょう | センサの声がうまく届きませんでした | もういちど |
| BLE切断 | つなぎなおしましょう | デバイスとの接続が途切れました | 再接続する |
| 通信 | 電波の良い場所で | 通信がうまくいきませんでした | もういちど |
| 電池低下 | そろそろ充電を | デバイスの電池が少なくなっています | わかりました |

## 11. Splash / App Icon / Logo

- **ロゴ**: 肉球マーク(線画ではなく塗り) + ワードマーク「HydroPaw」
  (システム太字/字間-0.5)。タグライン「ことばにならないものを、ことばに。」
- **スプラッシュ**: 背景=紙色(dark=黒)。中央にPrimary Softの円面+Mizuhaの
  肉球、下にワードマーク。ローディング表示なし(起動は一瞬であるべき)。
- **アプリアイコン**: Mizuha #12766B の単色地に白の肉球(角丸はOS任せ)。
  グラデ・文字・縁取り禁止。ダーク/ティント両対応(iOS 18)。

## 12. Sound (将来実装の方向性)

家族はすでに生活音の中にいる。音は**水**のメタファで、小さく、丸く。

| 場面 | 方向性 |
|---|---|
| 測定開始 | 水滴一粒(高すぎない、~600Hz、120ms) |
| 測定完了 | 二音の上行(水面の輪、マリンバ系、静かな解決感) |
| 通知(そろそろ測定) | 単音+余韻。繰り返さない |
| 高め通知 | 音色は同族のまま半音下げる(別の怖い音を作らない) |

## 13. Accessibility

- **Dynamic Type**: 固定高さを避け最小高さで組む。2段階拡大まで破綻しない。
- **VoiceOver**: StatusRing=「(犬名)の今日の状態: 安定しています。タップで測定」。
  グラフ=要約文をそのままラベルに(「正常範囲内で推移しています」)。
  装飾(ハロー・帯)はexcludeSemantics。
- **色覚多様性**: 状態は常に 色+記号(✓/!)+語 の三重表現(実装済)。
  Success/Dangerは輝度差も確保。
- **片手操作**: 主要アクションは常に画面下半分。ホームのリングタップは
  CTAと等価(どちらでも始められる)。
- コントラスト: 本文4.5:1以上、Caption(Tertiary)は装飾情報のみに使う。

## 14. App Store

- **名前**: HydroPaw
- **サブタイトル**: 犬の呼気から、毎日の見守りを。
- **キャッチコピー**: ことばにならないものを、ことばに。
- **紹介文(冒頭)**:
  「犬は『お腹の調子が悪い』と言えません。HydroPawは、呼気に含まれる
  わずかな水素から腸のコンディションを読み取り、『今日は安定しています』
  という毎日のことばにして届けます。1回3分。散歩のあとの新しい習慣に。」
- **スクリーンショット構成(5枚)**:
  1. ホーム(安定の緑リング)+「1秒で、今日のようすが分かる」
  2. 測定中(呼吸リング)+「測定は、静かな3分の儀式」
  3. 結果+「終わったら、ことばで受け取る」
  4. 履歴(日誌)+「変化は、線とことばで残る」
  5. 詳細+「受診のときは、数値がそのまま証拠に」
- **注記**: 「本アプリは診断を行うものではありません」を説明文とアプリ内
  (結果画面脚注)の両方に明記。

## 16. 対応記録・再セルフレビュー

(実装コミットで全項目対応。再レビューの結果は§16bに追記)

### §16a 対応記録 (A1–A20)

全20項目を実装済み。検証方法と証跡:

- **A1/A2/A19**: 実スクリーンショット(outputs/webshots/50–53)で Mizuha
  #12766B(light)/#4FB8A5(dark) のリング・CTA・タブ・チャートを目視確認。
  安定リング(緑)とブランド面(ティール)は同系統に調和し、
  「平常時は世界に溶け、注意時だけ浮く」意図どおり。
- **A3–A6/A16**: `error_page.dart`(kind別の行動題+ボタン)、
  `splash_page.dart`(スピナーなし)、arb文言監査で確認。
- **A7/A8/A20**: `home_page.dart` の Semantics(リング=button+状態文+ヒント、
  チャート=要約文)をコードで確認。Flutter実行は Mac側 `verify_app.sh` 待ち。
- **A9/A10**: CTA=minHeight+padding、タブ=textScaler clamp(1.3)+ellipsis。
- **A11–A15/A17/A18**: 各画面コード+Webスクリーンショットで確認。

### §16b 再セルフレビュー (2巡目)

実スクリーンショット(50 light / 51 dark / 52 measuring)に対して実施。

| # | 指摘 | 判定・対応 |
|---|---|---|
| A21 | **dark の CTA が白文字×明ティール(#4FB8A5)でコントラスト約2:1** — WCAG AA 不合格。51_brand_home_dark.png で発見 | `onAccent` トークンを新設(light=白 / dark=#0E0F12)。Flutter: AppPalette+FilledButtonTheme+ホームCTA+各スピナー/アイコン、Web: `--on-accent`+`.btn.primary`。53_brand_home_dark_fix.png+computed style `rgb(14,15,18)` で修正確認 |
| A22 | 安定時、リング枠(緑)とCTA(ティール)の2色が並ぶ | 意図どおりと判定。状態色は「リングの縁」だけに現れ、ブランド色は「行動」に現れる。warn/danger時は対比が明確になる設計(§3) |
| A23 | 空状態darkの見出し「はじめての測定をしてみましょう」が2行目「う」1文字で折返し | 器の問題ではなく文言長。ただし1文字折返しは端末幅依存で常発ではないため保留。文言短縮案「はじめての測定をどうぞ」は温度が下がるため不採用 |

**判定**: A21 を修正済み。A22–A24 は設計意図どおり/保留(実害なし)。
Web側は tsc / vitest 3/3 / build / 実描画+computed style まで検証済み。
Flutter側のビルド・実行検証は Mac の `./tool/verify_app.sh --build` が必要
(サンドボックスにFlutter SDKなし — 既知の制約)。
