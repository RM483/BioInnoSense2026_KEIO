# ⑪ プロダクトデザイン — 「センサーツール」から「犬の健康プロダクト」へ

## 0. 最重要原則

**このアプリはセンサーデータを見るアプリではない。**
ユーザーが知りたいのは「今日、うちの犬は元気?」であり、答えは ppm ではなく言葉で返す。

- 初見3秒で「犬の健康管理アプリ」と分かること
- ホームに専門用語(ppm・温度・湿度・BLE・Mock)を出さないこと
- 数値→意味の変換は `HealthAssessment`(Dart) / `assessment.ts`(TS) に一元化し、
  UIがppmを直接解釈しないこと

## 1. 意味への変換 (insights ドメイン)

| 平均H2 | レベル | ホームの言葉 | 色 |
|---|---|---|---|
| < 10ppm | stable | 「今日は安定しています」 | green |
| 10–20ppm | slightlyElevated | 「少し高めです。様子を見ましょう」 | amber |
| ≥ 20ppm | elevated | 「高めの値が続いています」 | red |
| 記録なし | none | 「はじめての測定をしてみましょう」 | blue |

- 20ppmはヒト呼気水素検査で一般的な陽性基準(+20ppm)に合わせた暫定値。
- 傾向: 前回比 ±20% で「改善/上昇」、それ以内は「変わりません」。
- 24時間無測定で「そろそろ測定をおすすめします」。
- 添える一言は 促し > 傾向 > レベル別コメント の優先順。

## 2. 画面構成 (5タブ)

```
┌──────────────────────────────────────────────┐
│ 🏠 ホーム   … 状態の言葉 / ひとこと / 最終測定 / 無数値トレンド / CTA │
│ 🩺 測定     … 呼吸リング / 準備状態を人の言葉で / はじめる          │
│ 📈 履歴     … 推移チャート / 状態ラベルの一覧 → 詳細(数値はここ)     │
│ 🐶 愛犬     … プロフィール(写真・名前・犬種・体重・誕生日)           │
│ ⚙️ 設定     … デバイス / デモ / 言語 / ログアウト (技術情報の隔離先) │
└──────────────────────────────────────────────┘
```

タブバーは自作(Material BottomNavigationBar不使用): ヘアライン1本 +
CupertinoIcons + 選択色のみ。ラベル10.5pt。

## 3. 画面遷移図

```
Splash ─→ Login ─→ ┌───────── Shell (5タブ) ─────────┐
                    │ Home ──CTA──→ Measureタブ          │
                    │ Measure ─はじめる→ ⬈ fullscreen     │
                    │ History ─行タップ→ History Detail   │
                    │ Settings ─→ Connect(push)          │
                    └───────────────────────────────┘
                              ⬈ fullscreen (タブバー非表示)
        MeasuringSession ─終了→ Result ─ホームへ→ Home
              │ ✕キャンセル → 戻る      (保存は自動)
              └ エラー → ErrorPage
```

**測定はひとつながりの体験**: はじめる → (fade+scale 320ms) → 測定中
(自動開始・4秒周期の呼吸リング・状態の言葉が主役・ppmはキャプション) →
終了 → (自動保存) → 結果(「測定できました」+ 静かな統計) → ホーム(評価が更新済み)。

## 4. デザイン意図

- **静けさ**: 影・グラデーション・波紋を排し、ヘアラインと余白で構成。
  色は 背景/カード/文字3階層/状態色(緑・琥珀・赤)/アクセント青 のみ。
- **安心感**: ホームは「状態の言葉 + ひとこと + 最終測定時刻」だけ。
  トレンドは数値レス(高さと状態色のバー7本)。
- **数値の居場所**: ppm・波形・温湿度・電池は 履歴詳細/結果/設定 に整理。
  隠すのではなく「知りたい人が辿れる場所」に置く。
- **アニメーション**: 呼吸リング(3–4s ease-in-out)、状態語のクロスフェード
  (250–300ms)、遷移は fade+scale(0.97→1.0)。派手さは足さない。
  Webは prefers-reduced-motion を尊重。
- **タイポグラフィ**: システムフォント。LargeTitle 28–34/700/字間タイト、
  状態語 24–26/700、本文 15–16、キャプション 13。数値はタブラー等幅。

## 5. 参考にしたプロダクト(思想)

| プロダクト | 借りた思想 |
|---|---|
| Apple Health | グループ背景+白カード、Large Title、右軸チャート、数値の階層 |
| Apple Fitness | 「今日をどう過ごしたか」を言葉とリングで返す要約思想 |
| Oura / Withings | スコアではなく状態フレーズ+一言コメントの安心設計 |
| Gentler Streak | 責めないトーン(「様子を見ましょう」)、休むことを許す文言 |

コピーではなく、余白・優先順位・言葉のトーンの考え方を移植した。

## 6. Web (webapp/)

同じ思想の単一カラム体験。ホーム=状態の言葉+履歴、測定中のみライブビュー
(状態語 + 小さなppm + 1本の線)。温湿度・電池・プロバイダは最下部「詳細」
カードへ隔離。MockProvider/BleProvider構成は不変(providers/index.tsの1箇所)。

## 7. 既知の改善点 (次の一手)

1. ホームのトレンドを週/月表示に拡張(Health風の期間セグメント)
2. 食事・症状記録(records ドメインは定義済み)と測定の重畳表示
3. 結果画面から「メモを残す」(measurement.note は保存対応済み)
4. 通知(そろそろ測定/高値が続いた時)— FCM拡張点はFunctionsに用意済み
5. ハプティクス(測定開始/完了時の軽いフィードバック)
6. 愛犬タブの多頭対応(DBは対応済み、selectedDogProviderの切替UIのみ)
7. オンボーディング(初回起動時の3枚: 何を測る/どう使う/何が分かる)
8. 呼気水素の医学的基準値は獣医学的エビデンスに合わせて要調整
   (現在はヒト基準20ppmの暫定運用 — insightsドメイン1箇所で変更可能)
